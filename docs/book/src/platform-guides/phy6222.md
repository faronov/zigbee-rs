# PHY6222 / PHY62x2

The repository contains a pure-Rust PHY62x2 radio backend and a
`thumbv6m-none-eabi` sleepy end-device example. The backend currently has
compile-time and host-test coverage only; no PHY6222 board result is claimed.

## Current support level

| Area | Status |
|---|---|
| Cortex-M0 firmware build | Compile-tested |
| PHY6 ROM-loader image | Generated and statically checked |
| GPIO, ADC, I2C | Source-level implementation |
| SPIF program/erase | RAM-resident, bounded waits |
| Crash-safe Zigbee security state | Implemented |
| Radio scan/join/data path | Awaiting hardware proof |
| AON system sleep and retention wake | Disabled in the example |
| PHY6252 | Separate NV bounds, otherwise unverified |

## Boot contract

PHY62x2 does not boot a generic Cortex-M ELF or a flat binary directly. The
public ROM-compatible layout reserves jump/configuration data below the
application SRAM entry and uses a segmented `PHY6` image:

```text
0x1fff0000..0x1fff03ff  application IRQ table used by ROM dispatch
0x1fff0400..0x1fff1837  ROM configuration/startup area
0x1fff1838              loaded initial-SP/Reset descriptor and run address
0x11010100              XIP application start
0x00010000              flash write address of the PHY6 header
```

`examples/phy6222-sensor/phy6222.x` emits the 256-entry ROM interrupt dispatch
table at `0x1fff0000`, places the two-word initial-SP/Reset descriptor at the
SRAM run address, and places executable text at the XIP address. Mutable
sections start after the run descriptor, so they cannot overwrite ROM-owned
memory.

## Build and package

```bash
cd examples/phy6222-sensor
cargo build --release
cd ../..

ELF=examples/phy6222-sensor/target/thumbv6m-none-eabi/release/phy6222-sensor
OBJCOPY=$(find "$(rustc --print sysroot)" -name llvm-objcopy | head -1)
"$OBJCOPY" -O ihex "$ELF" "$ELF.hex"
sh examples/phy6222-sensor/check-layout.sh "$ELF"
cargo run --quiet -p phy62x2-image -- "$ELF.hex" "$ELF.phy6.bin"
```

The layout check requires:

- `.jump_table` at `0x1fff0000`;
- `.vector_table` at `0x1fff1838`;
- `.text` at `0x11010100`;
- flash program and erase entry points in SRAM.

The image tool writes a 0x100-byte `PHY6` segment table, records SRAM and XIP
load addresses and CRCs, and appends the ROM image CRC. Flash the resulting
`.phy6.bin` at offset `0x10000` using a PHY62x2-aware programmer.

## Flash and security persistence

XIP must be bypassed while the external flash is programmed or erased.
`phy6222-hal::flash` therefore places the complete critical operation and all
SPIF waits in `.data.ram_code`, disables interrupts for the cache-bypass
window, and returns explicit timeout errors. The example wraps journal writes
with AON-RTC time compensation so the masked SysTick interval is not lost.

The sensor uses `SecurityStateJournal`, not the legacy item-by-item NV format.
The journal preserves network identity, keys, and exclusive outgoing frame
counter bounds across interrupted writes.

| Chip feature | Sector A | Sector B |
|---|---:|---:|
| `phy6222` | `0x7e000` | `0x7f000` |
| `phy6252` | `0x3e000` | `0x3f000` |

Only one chip feature may be selected.

## Sleep behavior

The example is a Zigbee sleepy end device at the MAC level: it turns the radio
off between polls and uses a 250 ms fast-poll interval after commissioning,
then a 10 second steady-state interval.

It does **not** enter AON system sleep. A system-sleep wake restarts through
ROM, so production support needs a hardware-proven startup/retention contract
that does not re-announce or rejoin the network after every poll interval.
Current-consumption and battery-life numbers should not be inferred until that
path is measured on hardware.

## Hardware validation order

1. Boot the packaged image and confirm vector/IRQ dispatch.
2. Verify RF initialization and channel calibration.
3. Capture Beacon Request, association, ACK, and Data Request frames.
4. Complete a fresh secured ZHA join and interview.
5. Reset during journal updates and prove counter monotonicity.
6. Add AON sleep only after flash wake and retained runtime state are proven.
